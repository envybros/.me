## **개요**

구조체는 관련된 여러 값을 하나의 단위로 묶는 사용자 정의 자료형이다. 객체 지향 언어에 익숙한 사람들에게는 구조체가 객체의 데이터 속성과 비슷하게 느껴질 수 있다. 이 장에서는 튜플과 구조체를 비교하고, 데이터를 그룹화하는 데 있어 구조체가 어떻게 더 나은 선택일 수 있는지 살펴볼 것이다.

구조체의 정의와 인스턴스화 방법, 구조체 타입과 관련된 동작을 정의하는 방법에 대해 알아볼 것이다. 특히, 구조체와 관련된 함수들인 메서드에 대해서도 논의할 것이다. 구조체와 열거형은 프로그램에서 새로운 타입을 만드는 데 있어 기본적인 역할을 하며, Rust의 강력한 컴파일 타임의 타입 검사 기능을 활용하는 데 도움을 준다.

## **구조체 정의 및 인스턴스 생성**

구조체는 "[튜플 타입](https://www.envys.me/posts/data-type/#%ED%8A%9C%ED%94%8C-%ED%83%80%EC%9E%85)" 섹션에서 논의된 튜플과 유사하게 여러 관련 값들을 담는다. 튜플과 같이 구조체의 각 부분은 서로 다른 타입이 될 수 있으나, 구조체는 각 데이터 조각에 이름을 부여하여 값의 의미를 더욱 명확히 한다. 이러한 이름 부여로 인해 구조체는 튜플보다 더 유연하고 의미가 명확하다. 즉, 인스턴스의 값에 접근하거나 지정할 때 데이터의 순서에 의존하지 않아도 된다.

구조체를 정의하기 위해서는 `struct` 키워드를 사용하고, 구조체 전체에 이름을 붙인다. 구조체의 이름은 그룹화된 데이터 조각들의 중요성을 나타내야 한다. 이후 중괄호 안에서 데이터 조각들의 이름과 타입을 정의하는데, 이들을 **필드**라고 부른다. 예를 들어, 아래 코드는 사용자 계정에 대한 정보를 저장하는 구조체를 보여준다.

파일명: *src/main.rs*

```rs
// User 구조체 정의

struct User {
    active: bool,
    username: String,
    email: String,
    sign_in_count: u64,
}
```

구조체를 정의한 후에는 각 필드에 구체적인 값을 지정하여 해당 구조체의 **인스턴스**를 생성해야 한다. 구조체의 이름을 언급한 다음, *key: value* 쌍을 포함하는 중괄호를 추가하여 인스턴스를 만든다. 여기서 `key`는 필드의 이름이고, `value`는 해당 필드에 저장할 데이터이다. 구조체에서 선언한 순서대로 필드를 지정할 필요는 없다. 즉, 구조체 정의는 타입의 일반적인 템플릿이며, 인스턴스는 특정 데이터로 이 템플릿을 채워 타입의 값을 생성한다. 예를 들어, 아래 코드는 특정 사용자를 선언하는 방법을 보여준다.

파일명: *src/main.rs*

```rs
// `User` 구조체의 인스턴스 생성

fn main() {
    let user1 = User {
        active: true,
        username: String::from("someusername123"),
        email: String::from("someone@example.com"),
        sign_in_count: 1,
    };
}
```

구조체의 특정 값에 접근할 때는 점 표기법을 사용한다. 예를 들어, 사용자의 이메일 주소에 접근하기 위해선, `user1.email`과 같이 표현할 수 있다. 인스턴스를 변경할 수 있는 경우에는, 점 표기법을 이용하여 필드의 값을 업데이트할 수 있다. 아래의 코드 예시에서는 `User` 인스턴스의 `email` 필드 값을 변경하는 방법을 보여준다.

파일명: *src/main.rs*

```rs
// `User` 인스턴스의 `email` 필드 값을 변경

fn main() {
    let mut user1 = User {
        active: true,
        username: String::from("someusername123"),
        email: String::from("someone@example.com"),
        sign_in_count: 1,
    };

    user1.email = String::from("anotheremail@example.com");
}
```

Rust에서는 인스턴스 전체가 변경 가능해야 하며, 특정 필드만을 변경 가능하게 설정하는 것은 허용하지 않는다. 함수의 마지막 표현식을 통해 새로운 구조체 인스턴스를 생성하고, 이를 암시적으로 반환하는 것도 가능하다.

다음은 사용자로부터 이메일과 사용자 이름을 입력받아 새로운 `User` 인스턴스를 반환하는 `build_user` 함수의 예시이다. 이때, `active` 필드는 항상 `true`를, `sign_in_count`는 `1`을 기본값으로 가진다.

```rs
// 이메일과 사용자 이름으로 `User` 인스턴스를 반환하는 `build_user` 함수

fn build_user(email: String, username: String) -> User {
    User {
        email,
        username,
        active: true,
        sign_in_count: 1,
    }
}
```

함수의 매개변수와 구조체 필드가 같은 이름을 사용할 경우, Rust는 필드 초기값 단순화(field init shorthand)을 제공하여, 변수 이름을 반복하지 않고도 필드를 초기화할 수 있게 한다. 만약 구조체에 더 많은 필드가 있을 경우, 이 구문은 코드를 더욱 간결하게 만들어 줄 것이다.

## **필드 초기값 단순화 사용하기**

앞서 제시된 코드에서, 매개변수의 이름과 구조체 필드의 이름이 동일할 경우 필드 초기값 단순화 구문을 사용할 수 있다. 이 방법을 사용하면, `username`과 `email`을 반복해서 작성할 필요가 없어진다. 다음 예제에서는 이를 적용한 `build_user` 함수를 보여준다.

파일명: *src/main.rs*

```rs
// `username`과 `email` 매개변수가 구조체 필드와 같은 이름을 가지므로
// 필드 초기값 단순화를 사용하는 `build_user` 함수

fn build_user(email: String, username: String) -> User {
    User {
        email,
        username,
        active: true,
        sign_in_count: 1,
    }
}
```

이 예제에서는 `User` 구조체의 새 인스턴스를 생성한다. 여기서 `email`이라는 필드에 `build_user` 함수의 `email` 매개변수 값을 할당한다. `email` 필드와 `email` 매개변수가 같은 이름을 가지므로, `email: email` 대신 단순히 `email`만 작성하여 필드 값을 설정할 수 있다. 이런 방식은 코드를 더 간결하고 명료하게 만들어 준다.

## **다른 인스턴스의 값으로 새 인스턴스 만들기**

Rust 프로그래밍에서는 기존 인스턴스의 대부분의 값을 그대로 사용하면서 일부만 변경해 새로운 인스턴스를 만드는 경우가 자주 있다. 이럴 때 구조체 업데이트 구문을 활용하면 꽤 편리하게 할 수 있다.

예를 들어, 아래의 예제는 `user2`라는 새로운 `User` 인스턴스를 만드는 방법을 보여준다. 이 과정에서 `user1`의 값을 대부분 그대로 사용하되, `email`만 새롭게 설정한다.

파일명: *src/main.rs*

```rs
// `user1`의 대부분 값을 그대로 사용하여 `user2` 인스턴스 생성, `email`만 변경

fn main() {
    // --생략--

    let user2 = User {
        active: user1.active,
        username: user1.username,
        email: String::from("another@example.com"),
        sign_in_count: user1.sign_in_count,
    };
}
```

구조체 업데이트 구문을 사용하면 더 간단하게 동일한 작업을 수행할 수 있다. 이 구문에서 `..` 기호는 지정하지 않은 필드들을 기존 인스턴스(`user1`)에서 가져와야 함을 나타낸다.

파일명: *src/main.rs*

```rs
// 구조체 업데이트 구문으로 `user1`의 값 중 일부만 사용하여 `user2` 생성

fn main() {
    // --snip--

    let user2 = User {
        email: String::from("another@example.com"),
        ..user1
    };
}
```

위 코드는 `user2`는 `user1`의 `username`, `active`, `sign_in_count` 필드 값을 그대로 유지하면서, `email` 값만 변경한다. `..user1` 구문은 이러한 필드들이 `user1`의 해당 필드 값을 사용함을 의미하며, `..` 구문은 코드의 마지막에 위치해야 한다.

구조체 업데이트 구문에서 `=` 기호를 사용하는 이유는 값을 "[이동](https://www.envys.me/posts/ownership/#%EB%B3%80%EC%88%98%EC%99%80-%EB%8D%B0%EC%9D%B4%ED%84%B0-%EC%9D%B4%EB%8F%99%EA%B3%BC%EC%9D%98-%EC%83%81%ED%98%B8%EC%9E%91%EC%9A%A9)"시키기 때문이다. 이 예제에서 `user2`를 만든 후에는 `user1`을 그대로 사용할 수 없게 된다. 왜냐하면 `user1`의 `username` 필드에 있던 `String` 값이 `user2`로 옮겨갔기 때문이다.

하지만 `user2`에 `email`과 `username`에 새로운 값을 설정했고, `user1`의 `active`와 `sign_in_count`를 사용했음에도, `user1`은 여전히 유효하다. 이는 `active`와 `sign_in_count`가 [복사](https://www.envys.me/posts/ownership/#%EC%8A%A4%ED%83%9D-%EC%A0%84%EC%9A%A9-%EB%8D%B0%EC%9D%B4%ED%84%B0-%EB%B3%B5%EC%82%AC)가 가능한 타입(`Copy` trait)이기 때문이다.

## **이름 없는 필드를 가진 튜플 구조체 사용하기**

Rust에서는 필드에 이름이 없고 타입만 있는 **튜플 구조체**를 제공한다. 이런 튜플 구조체는 튜플과 비슷하지만, 구조체 이름이 추가적인 의미를 부여한다. 튜플 구조체는 튜플에 이름을 부여해 다른 타입으로 만들고 싶을 때, 또는 각 필드에 이름을 지정하는 것이 필요 없을 때 유용하다.

튜플 구조체를 만들려면, `struct` 키워드와 구조체 이름을 적은 다음, 타입들을 나열하면 된다. 예를 들어, `Color`와 `Point`라는 두 튜플 구조체를 정의하고 사용하는 방법은 다음과 같다:

파일명: *src/main.rs*

```rs
struct Color(i32, i32, i32);
struct Point(i32, i32, i32);

fn main() {
    let black = Color(0, 0, 0);
    let origin = Point(0, 0, 0);
}
```

여기서 `black`과 `origin`은 각각 다른 튜플 구조체의 인스턴스로, 서로 다른 타입이다. 즉, `Color` 타입을 요구하는 함수에는 `Point` 타입을 사용할 수 없다. 하지만 이 구조체 인스턴스는 일반 튜플처럼 `.`을 사용하여 개별 요소에 접근할 수 있다.

## **필드가 없는 단위형 구조체**

필드가 전혀 없는 구조체도 만들 수 있다. 이런 구조체는 "단위형 구조체"라고 부르며, `()`, 즉 단위 타입과 비슷하게 작동한다. 단위형 구조체는 데이터를 저장하지 않지만, 특정한 타입에 트레잇을 적용할 때 유용하다. 예를 들어, AlwaysEqual이라는 단위형 구조체를 만들고 인스턴스화하는 방법은 다음과 같다:

파일명: *src/main.rs*

```rs
struct AlwaysEqual;

fn main() {
    let subject = AlwaysEqual;
}
```

`AlwaysEqual` 구조체를 만들려면 `struct` 키워드와 이름, 그리고 세미콜론을 사용하면 된다. 이 구조체는 특정 기능을 구현할 때 사용되며, 모든 인스턴스가 항상 동일하게 작동하도록 만들 수 있다. 이런 구조체는 주로 테스트 목적으로 활용된다. 추후에 이런 단위형 구조체에 트레잇을 정의하고 구현하는 방법을 다룰 예정이다.

## **구조체 데이터의 소유권**

```rs
struct User {
    active: bool,
    username: String,
    email: String,
    sign_in_count: u64,
}
```

Rust에서 구조체를 정의할 때 데이터의 소유권이 중요하다. 예를 들어, 위의 `User` 구조체에서는 `String` 타입을 사용하여 각 필드가 데이터를 소유하도록 했다. 이렇게 하면 구조체가 유효한 동안 해당 데이터도 유효하게 된다.

구조체가 다른 곳의 데이터에 대한 참조를 저장할 수도 있지만, 이때는 Rust의 **생명 주기(lifetime)** 기능을 사용해야 한다. 생명 주기는 구조체가 참조하는 데이터가 구조체가 존재하는 동안 유효하게 유지되도록 보장한다. 하지만 생명 주기를 지정하지 않고 참조만 저장하려고 하면 문제가 발생할 수 있다.

예를 들어, 아래 코드에서 `&str` 참조를 사용하는 경우는 컴파일되지 않는다:

파일명: *src/main.rs*

> 아래 코드는 컴파일되지 않는다.
{: .prompt-danger }

```rs
struct User {
    active: bool,
    username: &str,
    email: &str,
    sign_in_count: u64,
}

fn main() {
    let user1 = User {
        active: true,
        username: "someusername123",
        email: "someone@example.com",
        sign_in_count: 1,
    };
}
```

컴파일러는 이 코드에 생명 주기 지정자가 필요하다고 알려줄 것이다:

```bash
$ cargo run
   Compiling structs v0.1.0 (file:///projects/structs)
에러[E0106]: 생명 주기 지정자가 없음
 --> *src/main.rs*:3:15
  |
3 |     username: &str,
  |               ^ 명명된 생명 주기 매개변수가 필요함
  |
도움말: 명명된 생명 주기 매개변수 도입을 고려하십시오
  |
1 ~ struct User<'a> {
2 |     active: bool,
3 ~     username: &'a str,
  |

에러[E0106]: 생명 주기 지정자가 없음
 --> *src/main.rs*:4:12
  |
4 |     email: &str,
  |            ^ 명명된 생명 주기 매개변수가 필요함
  |
도움말: 명명된 생명 주기 매개변수 도입을 고려하십시오
  |
1 ~ struct User<'a> {
2 |     active: bool,
3 |     username: &str,
4 ~     email: &'a str,
  |

이 에러에 대한 자세한 정보는 `rustc --explain E0106`을 참조하십시오.
에러: 이전의 두 에러로 인해 `structs`를 컴파일할 수 없음
```

생명 주기 지정자 없이는 구조체에 참조를 저장하는 것이 허용되지 않는다. 이에 대한 해결 방법은 나중에 다루겠지만, 현재로서는 참조(`&str`) 대신 소유 타입인 `String`을 사용하여 이 문제를 피할 수 있다는 것을 알아두자.
